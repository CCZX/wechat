## 接口文档设计概述

文档描述了接口返回的数据格式，状态码，附加信息等

### API设计格式

`/api/v1/xxx`，确保符合RESTful规范。



### 状态码相关

>  总体的响应状态码：

| 状态码 | 描述                             |
| ------ | -------------------------------- |
| 2000   | 成功，一般在获取                 |
| 2001   | 失败，失败也指没有获取到响应数据 |
| 2002   | 用户                             |
| 2003   | 服务端错误                       |
| 2004   | 重复操作                       |
| 4001   | 权限不足，非法操作                       |

> USER相关的响应状态码

| 状态码 | 描述                               |
| ------ | ---------------------------------- |
| 1000   | 登录成功                           |
| 1001   | 登录失败：账号或密码错误           |
| 1002   | 验证码错误                         |
| 1003   | 用户已被注册                       |
| 1004   | 注册失败，可能是后台逻辑问题引起的 |
| 1005   | 注册成功                           |



### 总体返回格式：

```javascript
{
    status: Nnmber,// 状态码，
   	data: Object || String, // 数据
    msg: String, // 描述
}
```
